# 特性原子能力提取方法论

---

## 一、定义

特性原子能力是是从SE文档中提取的系统对外提供的**最小、独立、完整、不可再拆分**的业务能力单元，屏蔽技术实现细节，支撑用例设计、规则沉淀、覆盖分析、精准回归。

---

## 二、核心原则

| 原则 | 说明 |
|------|------|
| 只抽业务，不抽技术 | 屏蔽技术实现细节 |
| 最小不可拆，单一职责 | 一个原子只做一件事 |
| 无技术词、无实现、无架构 | 纯业务语言描述 |
| 可独立测试、可映射IBO | 支持测试用例生成 |
| 全链路可追溯、可治理 | 需求→特性→规则→用例 |

### 2.1 必须抽取的内容（✅ **必须抽取**）

- **各类业务校验**：唯一性、格式、范围、权限、关联依赖、业务规则校验、重复性拦截
- **后台自动业务处理**：自动状态变更、自动补全、自动计算、数据清洗、格式标准化
- **数据治理类动作**：数据关联绑定、关系解绑、历史归档、数据快照、台账维护
- **安全 & 审计 & 管控**：数据脱敏、访问拦截、操作留痕、权限控制、数据隔离
- **跨模块联动**：数据同步、业务消息流转、上下游联动、状态联动更新
- **异常兜底能力**：容错处理、数据修复、规则兜底、冲突消解

### 2.2 绝对不抽的内容（❌ 直接过滤）

严禁提取技术实现细节，以下词汇及概念直接过滤：

- 接口定义、字段、数据库表、缓存、MQ、微服务、事务、线程、部署、硬件、算法代码、参数入参出参、技术架构
- 禁止词汇：后台、接口、服务、数据库、字段、缓存、SQL、定时任务

## 三、提取方式

### 3.1 提取的字段

#### 必选字段

| 字段 | 取值 | 描述 | 提取方式 |
|-----|------|------|------|
| atom_id | str | 原子ID | 模型生成，格式为 `feat_atom_{1,2,3,...}`，编号数值从1开始递增 |
| atom_name | str | 原子名称 | 模型生成，对原子能力的一句话概括 |
| actor | str | 执行角色 | 模型提取 |
| subject | str | 关联主体 | 模型提取 |
| entity | str | 业务实体 | 模型提取 |
| entity_attrs | list[str] | 实体属性 | 模型提取 |
| behavior | str | 业务行为 | 模型提取 |
| pre_condition | str | 前置条件 | 模型提取，执行前必须满足的业务条件 |
| expected_result | str | 预期结果 | 模型提取，预期的输出 |
| ref_l3_rules | list[dict] | 关联L3规则 | 模型提取，参考《原子规则提取方法论》 |
### 3.2 提取步骤

1. 扫描全文，识别特性原子能力。

2. 针对识别出的每个原子能力：

   2.1. 生成atom_id；
   2.2. 总结atom_name；
   2.3. 按照方法论提取剩余字段
